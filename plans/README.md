## Dependencies

None — all four plans touch disjoint sets of files and can be executed in any order, sequentially or in parallel, by the same or different sessions.

## Recommended execution order

By increasing size/risk, smallest and lowest-risk first:

1. **003** — one file, ~10 new lines, fixes a real (if narrow-audience) correctness bug. Good first/quick win.
2. **004** — one file, one line changed. Trivial to execute and verify.
3. **002** — one file, moderate complexity (grid-rows animation technique is new to this codebase).
4. **001** — four files, the largest of the four, repeats the same fix pattern across each. Do this last so any lessons learned from 002/003 (e.g. how the `data-[state=...]` Tailwind variant reads in this project) carry over.

## Not selected (for future reference)

The audit surfaced 10 additional MEDIUM/LOW findings not turned into plans this round: unscoped `transition-all` on hover-lift cards (13 occurrences, 6 files) and its duplicated-recipe cohesion angle, width-animated pagination dots in `MascotIntro.tsx`/`GalleryCarousel.tsx`, zero press-feedback (`:active` states) sitewide, a blanket `prefers-reduced-motion` rule in `globals.css` that strips opacity/color feedback along with movement, `.page-transition`'s 520ms duration on every navigation, missing shared easing/duration tokens, `mascot-wiggle`'s 700ms click-feedback duration, `ReleaseCountdown.tsx` animating `height`/`width` instead of `transform`, and an abrupt grid/list view-toggle swap in `BlogList.tsx`/`MateriList.tsx`. Plus two missed-opportunity suggestions: no `@media (hover:hover)` guard anywhere sitewide (risk of sticky-hover on touch), and `SearchBox.tsx` reusing the scroll-triggered `Reveal` for a live-filtered results list. Re-run `/improve-animations` and ask for these if you want plans written for them.
